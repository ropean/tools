# QingTools — Collaboration Guide (for Claude)

Static tools site built with **Astro**. Each tool is a standalone page; the home
page is a searchable, category-grouped catalog. Deployed to Cloudflare Pages.

## Commands

```bash
pnpm dev        # Astro dev server (HMR) at http://localhost:4321
pnpm build      # Production build → _site/ (one .html per page + _astro/ assets)
pnpm preview    # Build, then serve _site with astro preview
pnpm ship       # Build + deploy to Cloudflare Pages (project: tools)
pnpm ship:ci    # Build + deploy to Cloudflare Pages (project: tools-ci)
```

Always run `pnpm build` after changes and confirm it completes with no errors.

## Project layout

```
src/
  data/tools.js          # Single source of truth: CATEGORIES + TOOLS + helpers
  layouts/
    Base.astro           # HTML shell: <head>, theme (no-flash), header, footer,
                         #   global clear-input wiring
    ToolLayout.astro     # Per-tool chrome: breadcrumb, "all tools", tool switcher,
                         #   sidebar (about + related); favorite hearts in switcher
  components/
    Header.astro         # Brand, nav, theme toggle, mobile menu
    Footer.astro
    CategoryIcon.astro   # Category glyphs (text/dev/web/db/media/util)
    ToolIcon.astro       # Card icon: favicon (external) / named icon / category icon
    ToolCard.astro       # Home card (icon, badge, title, desc, favorite heart)
  pages/
    index.astro          # Home: hero, search, category chips, Favorites, groups
    about.astro
    <id>.astro           # One page per live tool (uses ToolLayout)
    [id].astro           # Placeholder pages for ready:false tools (none currently)
  styles/theme.css       # Design tokens (light + dark) + all shared styles
public/                  # Served at site root (favicon.svg, etc.)
astro.config.mjs         # outDir=_site, build.format='file', dev /api proxy
```

Build output goes to `_site/` (git-ignored). `build.format: 'file'` emits one
standalone `.html` per page (e.g. `/about.html`, `/json-tree-viewer.html`).

## The tool catalog (`src/data/tools.js`)

Every tool is one entry in `TOOLS`:

```js
{ id, category, name, desc, ready, icon?, external?, favicon? }
```

- `ready: true` → the tool has a native page at `src/pages/<id>.astro`.
- `ready: false` → shows a "Coming soon" badge; a placeholder is generated by `[id].astro`.
- `external: 'https://…'` → off-site tool: card links out in a new tab, marked
  "External ↗", **no in-site page is generated**.
- `icon: 'name'` → named glyph for an internal tool (see `ToolIcon.astro`:
  globe/server/shield/file/shuffle/braces/code/qr/clock…).
- `favicon` → explicit card icon for external tools: a filename resolved against
  the site origin (`'favicon.svg'`, `'favicon.ico'`), a full URL, or the alias
  `'github'`. Without it, the category icon is used.

`CATEGORIES` defines category keys + labels. `groupByCategory()` is the single
sorted source: **categories sorted by label, tools by name** — the home chips,
home sections, tool switcher, and related list all derive from it, so ordering
is always consistent. "Favorites" is not a category; it is a client-side group
(localStorage) pinned first on the home page.

Descriptions: one sentence, verb-first, no parentheticals/domains/long lists.
Titles: the real product/repo name, first letter capitalized.

## Adding a tool

1. Add an entry to `TOOLS` in `src/data/tools.js`.
2. For a **live** tool: create `src/pages/<id>.astro` using `ToolLayout`. Copy the
   structure of an existing page (e.g. `random-string-generator.astro`):
   ```astro
   ---
   import ToolLayout from '../layouts/ToolLayout.astro';
   import { TOOLS } from '../data/tools.js';
   const tool = TOOLS.find((t) => t.id === '<id>');
   ---
   <ToolLayout tool={tool}> …UI… </ToolLayout>
   <style> …tool-specific CSS… </style>
   <script> …browser logic… </script>
   ```
   For an **external** tool: just set `external` (and optionally `favicon`) — no page.
3. `pnpm build` and confirm the page/card appears and ordering is correct.

## Conventions

- **English only** in all pages, code, and comments.
- Use theme tokens and shared classes; do not hardcode colors. Palette lives in
  `src/styles/theme.css` (`--bg`, `--fg`, `--muted`, `--border`, `--border-strong`,
  `--dark`, `--on-dark`, `--subtle`, `--ring`, `--error`, `--radius`, `--mono`, …).
- Shared classes: `.btn`, `.btn-primary`, `.btn-ghost`, `.btn-copy` (use for Copy),
  `.field`, `.field.mono`, `.panel`.
- **Astro scoped-style gotcha**: scoped styles compile to `.cls[data-astro-cid]`,
  which does NOT match elements created via `document.createElement`. If a tool
  builds DOM in JS (trees, lists, dynamic cards), mark its `<style>` `is:global`.
- Dark mode: styles must read tokens so they adapt via `[data-theme='dark']`.
  Monochrome marks that vanish in dark can invert (see `.ti-invert-dark`).

## Design

Warm "paper" theme documented in `design/lovable-DESIGN.md` (cream `#f7f4ed`,
charcoal `#1c1c1c`, opacity-derived grays, `#eceae4` borders, inset-shadow dark
buttons, 6px/12px radii, weights 400/600). Keep to it; avoid saturated accents.

## Deploy

- **CI**: `.github/workflows/deploy-to-cloudflare.yml` runs on `ci*`/`release*`
  branches (and manual dispatch): `pnpm install && pnpm build`, then
  `wrangler pages deploy ./_site`. `ci*` → project `tools-ci` (tools.aceapp.dev);
  `release*` → project `tools` (tools.ropean.org).
- **Local**: `pnpm ship` / `pnpm ship:ci` (requires a wrangler login).
